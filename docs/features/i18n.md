# i18n Feature

**Status:** Complete
**Phase:** 5
**Implements:** Locale config, currency/date formatters, translation catalogs

---

## Overview

Finansas uses a custom zero-dependency i18n system. There is no runtime locale switching — the locale is fixed at startup via environment variables. Translations live in plain TypeScript files, which gives full type-safety without a build step.

---

## Files

| File | Responsibility |
|------|---------------|
| `src/lib/locale.ts` | Reads env vars, exports `locale`, `t()`, `formatCurrency()`, `formatDate()`, `formatDateOnly()` |
| `src/lib/messages/en.ts` | English messages (source of truth for the `Messages` type) |
| `src/lib/messages/pt-BR.ts` | Portuguese messages (`satisfies Messages`) |
| `src/hooks/use-locale.ts` | Client-side hook — re-exports everything from `locale.ts` |
| `src/lib/__tests__/locale.test.ts` | 9 unit tests |

---

## Environment Variables

| Variable | Values | Default | Effect |
|----------|--------|---------|--------|
| `NEXT_PUBLIC_LOCALE` | `en`, `pt-BR` | `en` | Selects message catalog + number/date locale |
| `NEXT_PUBLIC_CURRENCY` | ISO 4217 code | `USD` | Currency for `formatCurrency()` |
| `NEXT_PUBLIC_DATE_FORMAT` | `mdy`, `dmy` | `mdy` | Month-day-year vs day-month-year |
| `NEXT_PUBLIC_TIME_FORMAT` | `12`, `24` | `12` | 12h AM/PM vs 24h |

## `locale.ts` API

```ts
export const locale = {
  lang: 'en' | 'pt-BR',
  currency: 'USD' | ...,
  dateFormat: 'mdy' | 'dmy',
  timeFormat: '12' | '24',
}

export function t<S extends keyof Messages>(section: S): Messages[S]
export function formatCurrency(amount: number): string
export function formatDate(date: Date | string): string      // date + time
export function formatDateOnly(date: Date | string): string  // date only
```

All functions are pure — they read env vars once at module load. Safe to call from Server Components, route handlers, and client components.

## Type System

`en.ts` uses `as const` to get literal types, then defines `Messages` via `Loosen<T>`:

```ts
type Loosen<T> = {
  [K in keyof T]: T[K] extends (...args: infer A) => string
    ? (...args: A) => string
    : T[K] extends string ? string : Loosen<T[K]>
}
export type Messages = Loosen<typeof messages>
```

`Loosen` converts string literals to `string` while preserving function signatures. This allows `pt-BR.ts` to use `satisfies Messages` — TypeScript validates the shape without narrowing to the English literal types.

## Adding Translations

1. Add the key to `src/lib/messages/en.ts` (inside the `messages` object, before `as const`)
2. TypeScript will error on `pt-BR.ts` until the same key is added there
3. Use in server context: `import { t } from '@/lib/locale'` → `t('section').key`
4. Use in client context: `const { t } = useLocale()` → `t('section').key`

---

## `use-locale.ts`

```ts
'use client'
import { locale, t, formatCurrency, formatDate, formatDateOnly } from '@/lib/locale'
export function useLocale() {
  return { ...locale, t, formatCurrency, formatDate, formatDateOnly }
}
```

Thin wrapper — no state, no effects. Re-exports the module-level values so client components get a hook-shaped API.
